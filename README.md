# Believe-It-or-Not

Командный проект по карточной игре «Верю — не верю» на Python 3.12.

---

## Описание проекта

`Believe-It-or-Not` — многопользовательская терминальная карточная игра,
реализованная по клиент-серверной архитектуре.

Один процесс запускает асинхронный TCP-сервер, а игроки подключаются к нему
через отдельные терминальные клиенты. Каждый клиент регистрируется под
уникальным именем и отправляет серверу текстовые команды.

Игра реализует правила карточной игры «Верю — не верю». Игрок кладёт одну или
несколько карт в стопку и объявляет их номинал. Следующий игрок выбирает:
поверить предыдущему игроку и добавить свои карты или не поверить и проверить
одну карту из последнего хода.

Если проверенная карта соответствует объявленному номиналу, стопку забирает
проверяющий игрок. Если карта не соответствует объявленному номиналу, стопку
забирает предыдущий игрок. Игрок, который забрал стопку, пропускает следующий
ход.

Проект поддерживает игру от 2 до 4 игроков, локализацию сообщений,
автоматическую сборку документации, запуск проверок через `doit` и сборку
дистрибутивов.

## Основные возможности

* терминальный клиент на основе `cmd.Cmd`;
* асинхронный сервер на основе `asyncio.start_server`;
* регистрация игроков по уникальному имени;
* подключение нескольких клиентов к одному серверу;
* игра для 2–4 игроков;
* приглашение игроков командой `start`;
* принятие приглашения командой `yes`;
* таймер ожидания игроков на 10 секунд;
* досрочный старт при наборе четырёх игроков;
* раздача колоды из 36 карт;
* отображение карт с эмодзи мастей;
* автоматический сброс четырёх карт одного номинала;
* запрет автоматического сброса четырёх тузов;
* запрет объявления туза как номинала;
* игровые команды `rules`, `play`, `believe`, `not`;
* клиентские команды `locale`, `documentation`, `quit`;
* автодополнение номиналов и локалей;
* персональная локаль для каждого игрока;
* обработка отключения активного участника;
* тестирование через `unittest`;
* документация Sphinx;
* локализация через `gettext` и Babel;
* автоматизация через `doit`;
* сборка `wheel` и `sdist`.

## Сетевой протокол

Клиент подключается к серверу и первой строкой отправляет имя пользователя.

Если регистрация прошла успешно, сервер отвечает:

```text
OK
```

Если регистрация невозможна, сервер отправляет одну из ошибок:

```text
ERROR bad or busy username
ERROR server is full
ERROR game already started
```

После успешной регистрации клиент отправляет команды построчно. Каждая команда
завершается переводом строки.

## Команды клиента и сервера

### `start`

Начинает набор игроков.

```text
start
```

Игрок, который отправил команду, становится создателем набора и автоматически
входит в будущую партию. Остальные подключённые игроки получают приглашение.

Если набор уже идёт или игра уже началась, сервер отправляет сообщение об
ошибке.

### `yes`

Принимает активное приглашение.

```text
yes
```

Если приглашение активно, игрок добавляется в список участников. Если набралось
четыре игрока, партия начинается досрочно. Если за 10 секунд никто не
присоединился к создателю, игра не начинается.

### `rules`

Выводит правила игры.

```text
rules
```

Команда не принимает аргументы. Она доступна через игровую модель.

### `play`

Начинает новый кон и кладёт карты в стопку.

```text
play K 1 2
```

Формат команды:

```text
play <rank> <card numbers>
```

`rank` — объявляемый номинал. Номера карт соответствуют текущему списку карт,
который клиент получает от сервера.

Допустимые объявляемые номиналы:

```text
6 7 8 9 10 J D K
```

Туз `A` не может быть объявлен.

### `believe`

Игрок верит предыдущему игроку и добавляет свои карты в стопку.

```text
believe 3 4
```

Формат команды:

```text
believe <card numbers>
```

Объявленный номинал при этом не меняется.

### `not`

Игрок не верит предыдущему игроку и проверяет одну карту из последнего хода.

```text
not 2
```

Формат команды:

```text
not <card number>
```

Если проверенная карта совпала с объявленным номиналом, стопку забирает
проверяющий игрок. Если карта не совпала, стопку забирает предыдущий игрок.

### `locale`

Меняет локаль сообщений для текущего клиента.

```text
locale en
locale ru
locale ru-2
```

Коды локалей:

* `en` — сообщения без перевода;
* `ru` — каталог `ru_RU`;
* `ru-2` — каталог `ru_BY`.

### `documentation`

Открывает локальную HTML-документацию.

### `quit`

Завершает клиентскую сессию.

```text
quit
```

Если игрок отключается во время активной партии, сервер останавливает игру для
всех участников.

## Правила игры

В игре используется колода из 36 карт.

Номиналы:

```text
6 7 8 9 10 J D K A
```

Масти:

```text
♣️ ♦️ ♥️ ♠️
```

Количество игроков:

```text
2–4
```

При старте партии сервер создаёт игровую модель, перемешивает колоду и раздаёт
карты игрокам.

Количество карт зависит от числа игроков:

* 2 игрока — по 18 карт;
* 3 игрока — по 12 карт;
* 4 игрока — по 9 карт.

Перед ходом игрока автоматически проверяется наличие четырёх карт одного
номинала. Если такой комплект есть, он сбрасывается. Четыре туза автоматически
не сбрасываются.

Если игрок начинает новый кон, он использует команду `play` и объявляет
номинал от `6` до `K`.

Если в стопке уже есть карты, следующий игрок должен выбрать `believe` или
`not`.

Если игрок выложил последние карты, следующий игрок обязан использовать
команду `not`.

Игрок без карт считается завершившим игру. Порядок завершения сохраняется в
игровой модели.

Игра заканчивается, если:

* в активной игре остался один игрок с картами;
* среди оставшихся карт остались только тузы;
* активный участник отключился от сервера.

## Автоматизация через doit

Файл задач находится в корне проекта:

```text
dodo.py
```

В проекте определены задачи:

```bash
python3 -m doit test
python3 -m doit flake8
python3 -m doit pydocstyle
python3 -m doit html
python3 -m doit package_docs
python3 -m doit i18n_pot
python3 -m doit i18n_po
python3 -m doit i18n_mo
python3 -m doit i18n
python3 -m doit check
python3 -m doit wheel
python3 -m doit sdist
```

Назначение задач:

* `test` — запускает тесты `unittest`;
* `flake8` — проверяет стиль кода;
* `pydocstyle` — проверяет docstring в пакете `believe`;
* `html` — собирает HTML-документацию;
* `package_docs` — копирует HTML-документацию внутрь пакета;
* `i18n_pot` — извлекает переводимые строки;
* `i18n_po` — создаёт или обновляет `.po`-файлы;
* `i18n_mo` — компилирует `.mo`-файлы;
* `i18n` — запускает полный цикл локализации;
* `check` — запускает основные проверки;
* `wheel` — собирает wheel-дистрибутив;
* `sdist` — собирает исходный дистрибутив.

Очистка сгенерированных файлов выполняется встроенной командой `doit`:

```bash
python3 -m doit clean
```

## Тестирование

В проекте используется `unittest`.

Запуск всех тестов:

```bash
python3 -m unittest discover -s tests
```

Запуск через `doit`:

```bash
python3 -m doit test
```

## Установка для разработки

Создать виртуальное окружение:

```bash
python3 -m venv venv
```

Активировать окружение на Linux/macOS:

```bash
source venv/bin/activate
```

Активировать окружение на Windows:

```bash
venv\Scripts\activate
```

Установить зависимости разработки:

```bash
python3 -m pip install -r requirements-dev.txt
```

Установить проект в editable-режиме:

```bash
python3 -m pip install -e .
```

Editable-установка создаёт консольные команды:

```bash
believe-server
believe-client
```

## Запуск приложения

Сначала запустить сервер:

```bash
python3 -m believe.server
```

или через установленный console script:

```bash
believe-server
```

Затем в другом терминале запустить клиента:

```bash
python3 -m believe.client ira
```

или:

```bash
believe-client ira
```
