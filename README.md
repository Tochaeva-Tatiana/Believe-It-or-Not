# Believe-It-or-Not

Командный проект по карточной игре «Верю — не верю» на Python 3.12.

---

## Описание проекта

Приложение реализует многопользовательскую терминальную карточную игру
«Верю — не верю».

Игра построена по клиент-серверной архитектуре. Один процесс запускает
асинхронный сервер, а игроки подключаются к нему через терминальные клиенты.
Каждый игрок регистрируется под уникальным именем и отправляет серверу
текстовые команды.

Первый игрок в кону кладёт одну или несколько карт и объявляет их номинал.
Следующий игрок должен решить, верит он предыдущему игроку или нет.

Если игрок верит, он добавляет в стопку свои карты и утверждает, что они имеют
тот же номинал. Если игрок не верит, он проверяет одну карту из последнего
хода предыдущего игрока. Если карта соответствует объявленному номиналу,
стопку забирает проверяющий. Если карта не соответствует объявленному
номиналу, стопку забирает предыдущий игрок.

Игра продолжается до выполнения условий завершения: когда в игре остаются
только тузы, когда игроки завершают игру по правилам игровой модели или когда
активный участник отключается от терминальной сессии.

Проект поддерживает локальную сетевую игру через TCP-сервер.

## Основные возможности

* Регистрация игроков по уникальному имени
* Подключение нескольких терминальных клиентов к одному серверу
* Игра для 2–4 игроков
* Команда `start` для начала набора игроков
* Команда `yes` для принятия приглашения
* Таймер ожидания игроков на 10 секунд
* Досрочный старт при наборе четырёх игроков
* Раздача 36 карт игрокам
* Отображение карт с эмодзи мастей
* Автоматический сброс четырёх карт одного номинала
* Запрет автоматического сброса четырёх тузов
* Запрет объявления туза в качестве номинала
* Команды `play`, `believe`, `not`, `rules`, `locale`, `quit`
* Автодополнение команд через `cmd.Cmd`
* Персональная локаль для каждого игрока
* Обработка отключения участника активной партии
* Тестирование через `unittest`
* Документация Sphinx
* Локализация через `gettext` и Babel
* Автоматизация проверок через `doit`
* Сборка wheel и sdist-дистрибутивов

## Архитектура

Проект разделён на три основные части.

### Клиент

Клиент находится в модуле:

```text
believe/client/__main__.py
```

Он отвечает за:

* запуск терминальной оболочки;
* чтение команд игрока;
* отправку команд серверу;
* получение сообщений от сервера;
* автодополнение игровых команд;
* открытие локальной документации.

Клиент не содержит игровой логики. Он только отправляет команды и выводит
сообщения, полученные от сервера.

### Сервер

Сервер находится в модуле:

```text
believe/server/__main__.py
```

Он отвечает за:

* запуск TCP-сервера;
* регистрацию клиентов;
* хранение подключённых игроков;
* обработку команд `start`, `yes`, `locale`, `quit`;
* передачу игровых команд в игровую модель;
* обработку событий игровой модели;
* рассылку личных и общих сообщений;
* завершение игры при отключении активного участника.

### Игровая модель

Игровая логика находится в модуле:

```text
believe/server/game.py
```

Она отвечает за:

* карты;
* колоду;
* игроков;
* раздачу;
* очередь ходов;
* автоматический сброс комплектов;
* команды `rules`, `play`, `believe`, `not`;
* проверку карт;
* определение победителей;
* завершение партии.

Игровая модель не работает с сокетами напрямую. Она возвращает события, которые
сервер затем отправляет нужным клиентам.

## Сетевой протокол

После подключения клиент первой строкой отправляет имя пользователя.

Если регистрация успешна, сервер отвечает:

```text
OK
```

Если регистрация невозможна, сервер отправляет сообщение об ошибке:

```text
ERROR bad or busy username
ERROR server is full
ERROR game already started
```

После успешной регистрации клиент отправляет команды построчно.

Пример:

```text
start
yes
rules
play K 1 2
believe 3 4
not 2
locale ru
quit
```

Одна строка соответствует одной команде.

## Игровые команды

### `start`

Начинает набор игроков.

```text
start
```

Игрок, отправивший команду, автоматически входит в состав партии. Остальные
подключённые игроки получают приглашение.

### `yes`

Принимает активное приглашение в игру.

```text
yes
```

Если в течение 10 секунд никто не присоединился, игра не начинается. Если
присоединился хотя бы один дополнительный игрок, партия начинается после
окончания таймера. Если набрались четыре игрока, игра начинается досрочно.

### `rules`

Выводит правила игры.

```text
rules
```

Команда доступна до начала партии и во время игры.

### `play`

Начинает новый кон. Игрок объявляет номинал и выбирает номера карт из своей
руки.

```text
play K 1 2
```

Допустимые объявляемые номиналы:

```text
6 7 8 9 10 J D K
```

Туз `A` не может быть объявленным номиналом.

### `believe`

Игрок верит предыдущему игроку и добавляет карты в стопку.

```text
believe 3 4
```

Номинал при этом не меняется: считается, что добавленные карты имеют тот же
номинал, который был объявлен в начале кона.

### `not`

Игрок не верит предыдущему игроку и проверяет одну карту из последнего хода.

```text
not 2
```

Если проверенная карта совпала с объявленным номиналом, стопку забирает
проверяющий игрок. Если не совпала, стопку забирает предыдущий игрок.

### `locale`

Меняет язык сообщений для текущего клиента.

```text
locale en
locale ru
locale ru-2
```

Поддерживаемые локали:

* `en` — английский язык;
* `ru` — основной русский перевод;
* `ru-2` — альтернативный русский перевод.

### `documentation`

Открывает локальную HTML-документацию.

```text
documentation
```

### `quit`

Завершает клиентскую сессию.

```text
quit
```

Если игрок выходит во время активной партии, игра завершается для всех
участников, а отключившийся игрок считается проигравшим.

## Правила игры

В игре используется колода из 36 карт.

Номиналы:

```text
6 7 8 9 10 J D K A
```

Масти отображаются с помощью эмодзи:

```text
♣️ ♦️ ♥️ ♠️
```

При старте партии карты перемешиваются и раздаются игрокам поровну.

Количество карт на игрока:

* 2 игрока — по 18 карт;
* 3 игрока — по 12 карт;
* 4 игрока — по 9 карт.

Перед ходом игрока проверяется наличие четырёх карт одного номинала. Если такой
комплект найден, он автоматически сбрасывается. Четыре туза не сбрасываются.

Первый игрок в кону выбирает объявляемый номинал и кладёт карты. Следующие
игроки выбирают: верить предыдущему игроку или проверить его.

Если игрок выложил последние карты, следующий игрок обязан выполнить проверку
командой `not`.

Игрок выигрывает, когда у него не остаётся карт на руках, но партия может
продолжаться для остальных игроков.

Игра заканчивается, когда выполняется одно из условий завершения игровой
модели, например когда в игре остаются только тузы или активная партия не может
продолжаться.

```text
start
```

```text

```


## Тестирование

В проекте используется только `unittest`.

Запуск всех тестов:

```bash
python3 -m unittest discover -s tests
```

Запуск отдельных тестовых модулей:

```bash
python3 -m unittest tests.test_game -v
python3 -m unittest tests.test_client -v
python3 -m unittest tests.test_server -v
```

Тесты сервера используют сокеты и таймауты, чтобы проверки не зависали.

Измерение покрытия не используется.

хранятся в `.po`-каталогах.

Поддерживаемые локали:

```text

```

Назначение локалей:

* `en` — английский язык по умолчанию;
* `ru` — основной русский перевод;

Смена локали в клиенте:

```text
locale en
locale ru
locale ru-2
```

Игровые команды, имена игроков, ранги и масти не переводятся.
